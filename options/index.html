Getting started

Switch domain extension adds links to the right click context menu. It uses simple regular expressions to enable you to match domains and replace sections. This extension was developed primarily for use when you are working on a large project with many test servers.

Setting Form

This setting form is a vanilla implementation of chrome-extension-options-form

Jasmine Spec Runner

Reset settings

To reset all the setting back to the default setting, click on the "Reset default options" button.

WARNING: this extension has no presets values, so resetting will just clear your settings.

Example usage

By default this extension loads with an example AEM setup above, where you have twelve deployments per stack (three each; author, publish, dispatcher and edge) in DEV, UAT and PRE-PROD environments.
There maybe any number of stacks depending on the number of development streams (e.g. dev1, dev2, dev3 etc...)

Given that you are viewing a page on the dispatcher on dev1 (web.dev1.foo-bar-baz.com), the links generated in the context menu would be to; web.uat1..., web.preprod1..., author.dev1..., publish.dev1... and www.incapsula.dev1... (i.e other web instances in the stack and other servers in the dev env. 


	author.dev1.foo-bar-baz.com:4502
	publish.dev1.foo-bar-baz.com:4503
	web.dev1.foo-bar-baz.com
	www.incapsula.dev1.foo-bar-baz.com

	author.uat1.foo-bar-baz.com:4502
	publish.uat1.foo-bar-baz.com:4503
	web.uat1.foo-bar-baz.com
	www.incapsula.uat1.foo-bar-baz.com

	author.preprod1.foo-bar-baz.com:4502
	publish.preprod1.foo-bar-baz.com:4503
	web.preprod1.foo-bar-baz.com
	www.incapsula.preprod1.foo-bar-baz.com
